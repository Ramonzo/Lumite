What is Lumite?

Lumite is a program for composing songs and sound effects in real time. You will be introduced to a series of orbital controls, their position and the way they are connected directly interferes with how the sound will behave, producing much more fluid and interactive sounds.
Lumite was made using Processing to develop the graphical interface and Pure Data as a sound engine.

Why Processing?

About the Software

Dynamics

There are types of connections that can be made, some elements only have outputs and others inputs and some have input and output. Everything works thinking about the flow that the sound will have, the path through which it will pass until reaching a digital-to-analog converter(DAC). This path, if thought of in a physical way, also includes the place through which the sound will pass, so if it "passes" through one part of the screen it produces a sound, on the other, another.

How it Works

The sound starts at an element, probably that element only has outputs, you can connect that output to as many inputs as possible. Then you connect that element to a digital-to-analog conversion element or to an effect. Don't forget, for the sound to be reproduced, the last stop must be a digital-to-analog converter (DAC). It is possible to connect oscillators, samples, instruments using the analog-to-digital converter (ADC), add effects and record. Currently you can work with 1 channel, but it is possible to add more channels by the source code (do this!).

Controllers

Each sphere represents an element of the software. Each element has its own controllers and each can be orbited to a position that represents how much it will interfere with the sound.
Just click and drag around. Each element may or may not have its position on the screen used to create interference in the sound, move the element across the screen and the sound changes in a fluid and very interesting way.

Connections

The connections work from A to B. That is, if you connect A to B you cannot connect B to A, but you can circulate with a third element, do A-> B-> C-> A. Beware of loops, loops cause an exponential increase in the wave and can cause problems with your equipment or recording, use this feature wisely.
Creativity in connections flies! Connect, move elements and experiment with different ways to make an interesting effect or sound. Try this with a microphone!
Press: Ctrl+Click - Click on element A and then on element B to make a connection.

Converters(DAC/ADC)

ADC -> Analog-to-Digital Converter:
This converter is used so that, through channel 1, you can insert a sound produced in real time into the software, test it with a guitar or microphone.
This converter only has outputs. Just like any output, this converter can create different connections.
DAC -> Digital-to-Analog Converter:
This converter receives a sound that is currently bits and converts it to your computer's analog output.
This converter only has inputs, it is the last connection point that your flow must have.

Recoder

The recorder is a receiver, that is, it has only inputs. As soon as a connection is made to the recorder it starts recording all the stream that reaches it.
Think about working with it side by side with the DAC.
The recorder has two volume controls (L / R) so you can reduce the gain in sound during recording. Popping sounds are common in recording.

Oscillators

Oscillators do some magic in the sound. The oscillator produces the sound of a sine wave that is played infinitely while it is connected to an input.
Oscillators only have outputs.
Controls include volume control (L / R) and oscillation frequency control.

Samplers

Samples are similar to oscillators, the difference being that they infinitely reproduce any sound that has been programmed for them.
The samples include volume control (L / R).

Effects

Effects are elements that have both inputs and outputs.
They can be connected to other effects and their controls do not necessarily have a pattern, each effect has its controls.
Currently you can only understand which control is what by checking them inside the Pure Data source code or following the documentation if available.
